## 1. 안내 화살표 (첫 진입 가이드) — `N02 메뉴 화면`

화면: 메뉴 화면 첫 진입 시 어두운 오버레이 + 말풍선 + 점선 화살표

- [ ] 점선과 화살촉 **색이 주황색**(#FF6B35)으로 통일됐는가 (이전: 노란색)
- [ ] 화살표 머리가 **비정상적으로 크지 않고** 적당한가
- [ ] 점선이 **직선이 아니라 살짝 곡선**으로 자연스럽게 이어지는가
- [ ] 화살촉이 대상 블록(매장/포장·마이크·대화기록·AI추천·상세칩)의 **경계를 정확히** 가리키는가 (블록 위에 파묻히지 않음)
- [ ] 추천 메뉴가 없을 때 "AI 추천 라벨" 말풍선이 자동으로 숨는가

> 가이드는 세션당 1회만 떠서, 다시 보려면: 브라우저 콘솔에서
> `sessionStorage.removeItem('n02GuideSeen')` 후 새로고침.

## 2. 상세페이지 하단 "장바구니에 담기" 버튼 — `N02 → 메뉴 썸네일 터치`

- [ ] 메뉴 카드 썸네일을 눌러 **상세 오버레이**를 연다
- [ ] 맨 아래 "장바구니에 담기 · ₩금액" 버튼이 **좌우 여백이 대칭**이고 우측에 붙거나 잘리지 않는가
- [ ] 아이콘 + 텍스트 + 금액이 **가운데로 묶여** 보이는가
- [ ] 버튼을 누르면 눌리는 효과(살짝 작아짐)가 있는가

## 4. 눈치 기능 (망설임 감지 → AI 추천) — `N02 메뉴 화면`

> FastAPI(AI 서버)가 켜져 있어야 추천 응답이 옵니다.

- [ ] 가이드를 닫고 **아무것도 담지 않은 채 25초 이상 가만히** 두면, 잠시 후 AI 추천 멘트(토스트)와 추천 메뉴 카드 강조가 뜨는가 (`silence` 신호)
- [ ] 메뉴를 **담지 않고 상세만 3번 이상** 열었다 닫으면 추천이 뜨는가 (`repeat_browse` 신호)
- [ ] 메뉴를 한 번 담으면 위 감지 카운터가 초기화되는가 (담은 뒤엔 과하게 안 뜸)
- [ ] 개입은 세션당 최대 3회까지만, 그리고 너무 자주 안 뜨는가 (쿨다운 45초)

> 콘솔에 `[Nunchi] 신호 발생 → silence` 같은 로그가 찍히는지로도 확인 가능.

## 5. 실제 영수증 출력 — `결제 완료(P05) 화면`

확인 경로: 메뉴 담기 → 결제하기 → 결제수단 선택 → (카드/바코드) → 완료 화면 → 출력 모달에서 "영수증 출력"

- [ ] 완료 화면에서 "영수증 출력" 선택 시 **실제 인쇄 대화상자(또는 프린터 출력)** 가 뜨는가
- [ ] 영수증에 **매장명 / 사업자번호 / 주문일시 / 품목 명세(메뉴·수량·금액) / 옵션 / 공급가·부가세 / 합계 / 결제수단 / 대기번호 / 바코드** 가 나오는가
- [ ] **한글이 깨지지 않는가** (□□□ 또는 ??? 없이 정상 출력) — UTF-8 인쇄 방식
- [ ] 영수증 폭이 영수증 용지(80mm)에 맞는가

> 프린터가 없으면 인쇄 미리보기에서 레이아웃만 확인하면 됩니다.
> 실제 매장 사업자 정보는 `js/common/receipt.js` 상단 `STORE_INFO` 에서 교체하세요.

## 6. 바코드 결제 (HID 스캐너) — `P07 바코드 화면`

- [ ] 결제수단에서 **카카오페이(바코드)** 선택 → 바코드 화면 진입
- [ ] (스캐너 있을 때) 손님 폰 바코드를 스캔하면 **그 값으로** 결제가 진행되는가
- [ ] (스캐너 없을 때) URL 뒤에 `?demo=1` 붙이면 2.5초 후 자동 성공 (동작 확인용)
- [ ] 45초간 스캔이 없으면 실패 화면으로 넘어가는가 (멈춤 방지)

> 스캐너는 키보드처럼 숫자를 빠르게 입력 후 Enter 를 보냅니다. 그 패턴을 잡아 처리합니다.
