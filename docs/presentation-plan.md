# NUNCHI KIOSK 발표 구성 계획서

> 핵심 전략: **"기능 나열"이 아니라 "차별점 증명"**.
> 한 슬라이드 = 한 메시지. 모든 장표는 "이게 기능 설명인가, 차별점 증명인가?"를 자문한다.
> 발표 뼈대: **문제 → 해결(눈치+MCP) → 데모 → 운영자 대시보드 → 향후 비전**

---

## 0. 발표 전체 흐름 한눈에

```
[1] 표지
[2] 문제 정의        ── 기존 키오스크의 한계 / 배리어프리
[3] NUNCHI 한 줄 정의 ── "자연어만으로 결제까지"
[4] 핵심 차별점 ①    ── 눈치 기능 (USP)          ★시간 최다 배분
[5] 핵심 차별점 ②    ── MCP 기반 AI 자율 제어
[6] 시스템 한눈에     ── 5계층 → 1장 압축 (30초)
[7] ▶ 시연영상 #1     ── 음성 자율주문 풀플로우    ★1차 클라이맥스
[8] 눈치 동작 원리    ── 감지신호 → 자동추천
[9] ▶ 시연영상 #2     ── 눈치 트리거 실제 장면
[10] 운영자 대시보드   ── 관리자+모니터링 통합 1장
[11] 결과/성과 지표    ── 숫자로 증명
[12] 향후 비전        ── MCP 개방 / 동국대 AI팩 / Claude
[13] ▶ 시연영상 #3     ── 4개국어 동시 주문 데모    ★최종 클라이맥스
[14] 마무리 / Q&A
[A] 부록             ── 기능요구사항 표·기술상세 (참고용)
```

## 1. 시연영상 배치 전략 (3개 지점)

| 위치 | 영상 | 길이 | 의도 |
|------|------|------|------|
| **[7] 본론 중반** | **음성 자율주문 풀플로우** (말 한마디 → 추천 → 담기 → 결제) | 60~90초 | 1차 클라이맥스. "진짜 된다"를 먼저 보여주고 뒤에서 원리 설명 |
| **[9] 눈치 설명 직후** | **눈치 트리거 장면** (머뭇거림 감지 → AI가 먼저 추천) | 30~45초 | USP를 글이 아닌 화면으로 증명 |
| **[13] 비전 직전·직후** | **4개국어 동시 주문** (한·영·중·일 4창이 한 엔진 제어) | 45~60초 | 최종 클라이맥스. MCP 개방성의 결정적 증거 |

> 배치 원칙: **영상은 "주장 → 증거" 순서**로 깐다.
> 차별점을 말로 먼저 선언하고, 바로 다음 장에서 영상으로 못 박는다.
> 영상은 무음 자막 버전을 준비해 발표자가 위에 멘트를 얹을 수 있게 한다.

## 2. 장표별 상세 (자료 목록 + 핵심 3줄)

### [1] 표지
- **자료**: 프로젝트명 "NUNCHI (눈치) KIOSK", 한 줄 슬로건, 팀명/학교, 키오스크 비주얼 1컷
- **핵심 3줄**
  1. LLM Agentic AI 기반 배리어프리 자율주문 키오스크.
  2. "음성 대화만으로 결제까지" 라는 슬로건을 표지에 명시.
  3. 첫 장부터 "차세대 키오스크"라는 인상을 비주얼로 각인.

### [2] 문제 정의
- **자료**: 기존 키오스크 불편 사례(복잡한 화면·고령자/장애인 배제), 통계/뉴스 인용 1~2개, 페인포인트 아이콘
- **핵심 3줄**
  1. 현 키오스크는 화면이 복잡하고 디지털 약자를 배제한다.
  2. 사용자가 "어디를 눌러야 할지" 스스로 알아내야 하는 구조.
  3. → 사용자가 시스템에 맞추는 게 아니라, 시스템이 사용자를 살피는 키오스크가 필요.

### [3] NUNCHI 한 줄 정의
- **자료**: 핵심 정의 문장, before/after 대비 이미지, 2가지 모드(일반 UI / 아바타 UI) 아이콘
- **핵심 3줄**
  1. 자연어 대화만으로 메뉴 탐색→추천→담기→결제까지 완료하는 완전 자율 주문.
  2. 터치(일반 UI) + 음성(아바타 UI) 두 모드를 병렬 지원.
  3. 두 개의 무기 — 먼저 살피는 "눈치", 직접 조작하는 "MCP".

### [4] 핵심 차별점 ① — 눈치 기능 ★
- **자료**: 감지신호 3종(체류시간·반복탐색·음성불확실성) 다이어그램, "AI가 먼저 말을 건다" 컨셉 이미지
- **핵심 3줄**
  1. 도움을 요청하지 않아도 시스템이 망설임을 먼저 감지한다.
  2. 체류시간·반복탐색·음성 불확실성 → 조건 충족 시 추천 자동 트리거.
  3. 프로젝트 이름이자 유일한 차별점. **발표 시간 최다 배분.**

### [5] 핵심 차별점 ② — MCP 기반 AI 자율 제어
- **자료**: MCP 도구 3종 표(DB / UI Control / Payment), "AI가 직접 화면을 클릭" 컨셉 컷
- **핵심 3줄**
  1. AI가 답만 하는 게 아니라 MCP로 화면을 직접 조작·담기·결제까지 실행.
  2. DB조회·UI제어·결제 3개 MCP Tool을 의도에 맞춰 자율 체이닝.
  3. 사용자 확인 모달로 안전장치를 둬 자율성과 통제를 양립.

### [6] 시스템 한눈에 (압축)
- **자료**: 5계층 아키텍처 1장(HW→Presentation→MCP AI→Backend→Infra), 통신 흐름 화살표만
- **핵심 3줄**
  1. React UI ↔ Spring Boot ↔ FastAPI(AI) 3축 + MCP 프로토콜.
  2. 박스를 낱개 설명하지 않고 30초 안에 전체 그림만 전달.
  3. 핵심은 "MCP가 모든 계층을 잇는 표준 통로"라는 점 하나.

### [7] ▶ 시연영상 #1 — 음성 자율주문 풀플로우 ★
- **자료**: 60~90초 데모영상(자막 버전), 영상 위 발표자 멘트 큐시트
- **핵심 3줄**
  1. 말 한마디로 추천→담기→결제까지 한 번에 도는 장면을 먼저 보여준다.
  2. "진짜 동작한다"를 각인시킨 뒤 원리 설명으로 넘어간다.
  3. 발표의 1차 클라이맥스 — 가장 임팩트 큰 순간.

### [8] 눈치 동작 원리
- **자료**: 트리거 플로우(신호 감지 → 판매데이터 기반 추천 → 제시 → 처리), 실시간 인기메뉴 추천 컨셉
- **핵심 3줄**
  1. 감지 신호가 임계치를 넘으면 추천 엔진이 자동 발동.
  2. 추천은 오늘의 실시간 판매 데이터를 기반으로 동적으로 산출.
  3. "주문 데이터 → 추천"으로 순환하는 구조 (운영자 대시보드 장표로 연결).

### [9] ▶ 시연영상 #2 — 눈치 트리거 실제 장면
- **자료**: 30~45초 영상(머뭇거림→AI 선제 추천), 전후 화면 캡처
- **핵심 3줄**
  1. 사용자가 머뭇거리자 AI가 먼저 추천을 띄우는 실제 장면.
  2. USP를 글이 아니라 화면으로 직접 증명.
  3. [8]에서 설명한 원리가 실제로 작동함을 못 박는다.

### [10] 운영자 대시보드 (관리자 + 모니터링 통합)
- **자료**: 대시보드 목업 1장 — 실시간 매출 / 오늘의 인기메뉴 / 진행 세션 / 시스템·AI 상태
- **핵심 3줄**
  1. 매장 운영(매출·인기메뉴·세션)과 시스템 상태(서버·AI·에러)를 한 화면에.
  2. 관리자 페이지와 모니터링을 하나로 합쳐 운영자 관점으로 단순화.
  3. **주문 데이터가 곧 추천 데이터로 순환** — 눈치 기능과 연결되는 핵심 메시지.

### [11] 결과 / 성과 지표
- **자료**: 핵심 숫자 카드(AI 응답 3초 이내 / 음성인식 90%+ / 터치·음성 병렬 충돌 0), 완성도 요약
- **핵심 3줄**
  1. 비기능 목표를 나열하지 않고 핵심 1~2개만 숫자로 증명.
  2. "터치와 음성이 동시에 들어와도 충돌 없음" 등 안정성 강조.
  3. MVP가 실제 동작 가능한 수준까지 완성됐음을 객관 지표로.

### [12] 향후 비전 — MCP 개방
- **자료**: 개방형 다이어그램 `[NUNCHI 엔진] ──MCP──< Claude / ChatGPT / 동국대 AI팩 >`
- **핵심 3줄**
  1. 엔진을 MCP 표준으로 감싸 특정 LLM에 종속되지 않는다.
  2. **2026년부터 동국대학교 AI 팩으로 제공**, Claude로도 주문 가능.
  3. 키오스크 한 대가 아니라 "AI가 붙는 주문 인프라"로 확장.

### [13] ▶ 시연영상 #3 — 4개국어 동시 주문 ★
- **자료**: 45~60초 영상(한·영·중·일 4창이 한 엔진 제어), 멀티윈도 화면
- **핵심 3줄**
  1. 4개 언어 창이 하나의 엔진을 동시에 제어하는 장면.
  2. MCP 개방성·확장성의 결정적 증거 — 최종 클라이맥스.
  3. "표준 프로토콜이라 가능한 일"이라는 메시지로 마무리.

### [14] 마무리 / Q&A
- **자료**: 핵심 3줄 요약, 팀 크레딧, 연락처/리포지토리
- **핵심 3줄**
  1. 문제(배리어프리) → 해결(눈치+MCP) → 비전(MCP 개방)을 한 장에 재진술.
  2. "사용자를 살피고, AI가 실행하고, 누구나 붙는다"로 압축.
  3. Q&A로 자연스럽게 전환.

### [A] 부록 (참고용 — 발표 본편엔 미포함)
- **자료**: 기능요구사항 ID 표(0-1~9-3), 음성 파이프라인 6단계, 통신구조 상세, SQL/기술스택 버전
- **용도**: 질의응답 대비용. 본편 슬라이드에는 절대 넣지 않는다.

## 3. 버릴 것 / 압축할 것 체크리스트

- [ ] 기능요구사항 ID 표 전체 → **부록으로 이동** (본편 삭제)
- [ ] 음성 파이프라인 6단계 낱개 → **1줄 화살표로 압축** ([6]에 흡수)
- [ ] 5계층 아키텍처 박스별 설명 → **1장 30초**로 압축

- [ ] 비기능 요구사항 나열 → **핵심 숫자 1~2개만** ([11])
- [ ] 관리자 페이지 + 모니터링 2장 → **운영자 대시보드 1장**으로 통합

## 4. 발표 운영 팁

- 영상은 모두 **무음 자막 버전** 준비 → 발표자가 위에 멘트를 얹는다.
- 시연영상 3개는 **"주장 → 증거"** 순서로, 차별점 선언 직후 배치.
- 눈치([4],[8],[9])에 전체 시간의 **30% 이상** 배분.
- 비전([12],[13])은 마지막에 배치해 **여운**으로 끝낸다.
